折腾了一段时间，终于基于 Hexo + GitHub Pages 把这个静态博客搭建起来了。

这不仅仅是一个网站，更是我整理技术体系、复盘面试经历的“根据地”。

### 为什么要建这个站？

最近在面试和学习的过程中发现，**输入再多，如果不输出，很快就会遗忘**。与其在碎片化的笔记里打转，不如把思考过程结构化地记录下来。

在这个博客里，我主要会更新以下几个方向的内容：

* **面试复盘**：真实面试中遇到的“坑”与反思（Java 后端、中间件、系统设计）。
* **源码分析**：不再浅尝辄止，深入探究框架底层的实现原理。
* **开源实战**：记录参与开源项目（Open Source）的过程，从 Good First Issue 到核心 PR 的进阶之路。

### 接下来要做的事 (To-Do List)

给自己定个小目标，保持持续交付（Continuous Delivery）的习惯：

- [ ] 输出第一篇高质量的面试复盘文章
- [ ] 寻找并解决第一个开源项目的 Issue
- [ ] 坚持每周至少一次的代码提交 (Commit)

### 保持联系

既然选择了技术这条路，唯有持续精进。欢迎通过 GitHub 关注我的动态，或者在 Issue 区与我交流。
